# Инструкция по деплою на Render

## Вариант 1: Автоматический деплой через render.yaml

1. **Подготовка репозитория:**
   - Убедитесь, что все изменения закоммичены и запушены в GitHub/GitLab/Bitbucket

2. **Создание сервиса на Render:**
   - Зайдите на [render.com](https://render.com)
   - Войдите в аккаунт или зарегистрируйтесь
   - Нажмите "New +" → "Blueprint"
   - Подключите ваш репозиторий
   - Render автоматически обнаружит файл `render.yaml` и настроит сервис

3. **Настройка переменных окружения (если нужно):**
   - В настройках сервиса добавьте необходимые переменные окружения
   - Например, если у вас есть API endpoints, добавьте их через Environment Variables

4. **Деплой:**
   - Render автоматически запустит сборку и деплой
   - После успешного деплоя вы получите URL вида: `https://your-app-name.onrender.com`

## Вариант 2: Ручная настройка Web Service

1. **Создание Web Service:**
   - Зайдите на [render.com](https://render.com)
   - Нажмите "New +" → "Web Service"
   - Подключите ваш репозиторий

2. **Настройки:**
   - **Name:** stellar-burgers (или любое другое имя)
   - **Environment:** Node
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm run start:server`
   - **Plan:** Выберите Free или другой план

3. **Переменные окружения:**
   - Добавьте переменные, если они нужны для вашего приложения
   - `NODE_ENV=production` (устанавливается автоматически)
   - `PORT` устанавливается автоматически Render'ом

4. **Создание сервиса:**
   - Нажмите "Create Web Service"
   - Render начнет сборку и деплой

## Вариант 3: Статический сайт (альтернатива)

Если вы хотите использовать статический сайт вместо Web Service:

1. **Создание Static Site:**
   - Нажмите "New +" → "Static Site"
   - Подключите репозиторий

2. **Настройки:**
   - **Build Command:** `npm install && npm run build`
   - **Publish Directory:** `dist`
   - **Single Page Application:** Включите эту опцию (важно для React Router!)

3. **Создание:**
   - Нажмите "Create Static Site"

## Примечания

- **Бесплатный план:** Приложения на бесплатном плане "засыпают" после 15 минут неактивности. Первый запрос после этого может занять до 30 секунд.
- **Переменные окружения:** Если ваше приложение использует переменные окружения (например, API URLs), добавьте их в настройках сервиса.
- **Автоматический деплой:** По умолчанию Render автоматически деплоит при каждом push в основную ветку (обычно `main` или `master`).

## Проверка деплоя

После успешного деплоя:
1. Откройте URL вашего приложения
2. Проверьте, что все маршруты работают корректно
3. Убедитесь, что API запросы работают (если используются)

## Troubleshooting

Если что-то не работает:
- Проверьте логи сборки в панели Render
- Убедитесь, что все зависимости установлены
- Проверьте, что переменные окружения настроены правильно
- Убедитесь, что порт настроен правильно (Render автоматически устанавливает PORT)
